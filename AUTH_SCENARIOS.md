# أمثلة سيناريوهات عملية لنظام التسجيل الذكي

## 🎯 السيناريو 1: المستخدم الأول

```
1. المستخدم يفتح التطبيق
   ↓
2. initAppWithSmartLogin() تتحقق من localStorage
   ↓
3. لا توجد جلسة سابقة
   ↓
4. اعرض شاشة تسجيل الدخول
   ↓
5. المستخدم يدخل المعرف وكلمة المرور
   ↓
6. handleLogin() يتحقق من قاعدة البيانات (Firebase)
   ↓
7. حفظ الجلسة والبيانات محلياً
   ↓
8. افتح التطبيق
```

---

## 🎯 السيناريو 2: المستخدم المسجل (مع إنترنت)

```
1. المستخدم يفتح التطبيق
   ↓
2. initAppWithSmartLogin() تتحقق من localStorage
   ↓
3. وجدنا جلسة سابقة ✅
   ↓
4. المستخدم متصل بالإنترنت ✅
   ↓
5. جاري المزامنة مع قاعدة البيانات
   ↓
6. تحديث البيانات المحلية بأحدث النسخة
   ↓
7. افتح التطبيق مباشرة بدون شاشة تسجيل
```

## 🎯 السيناريو 3: المستخدم المسجل (بدون إنترنت)

```
1. المستخدم يفتح التطبيق
   ↓
2. initAppWithSmartLogin() تتحقق من localStorage

3. وجدنا جلسة سابقة ✅
```

## 🎯 السيناريو 4: عودة الإنترنت

```
أثناء استخدام التطبيق:

1. المستخدم يضيف مريض جديد
   ↓
2. حفظ محلياً ✅
   ↓
3. عادت الإنترنت 🟢
   ↓
4. event 'online' يُطلق
   ↓
5. syncWithOnlineDatabase() يبدأ
   ↓
6. تحديث قاعدة البيانات بالتغييرات
   ↓
7. إشعار: ✅ تمت المزامنة بنجاح
```

## 🎯 السيناريو 5: تسجيل خروج

```
1. المستخدم يضغط "تسجيل خروج"
   ↓
2. تأكيد: "البيانات ستبقى محلياً"
   ↓
3. handleLogout() يمحو الجلسة
   ↓
4. تحديث البيانات المحلية محفوظ ✅
   ↓
5. إعادة تحميل الصفحة
   ↓
6. شاشة تسجيل دخول جديدة
```

---

## 🎯 السيناريو 6: جهازان مختلفان

```
الجهاز الأول (بـ checkbox "تذكر هذا الجهاز"):
- initAppWithSmartLogin() → افتح مباشرة
- rememberDevice = true ✅
- صلاحية مدى الحياة (30 يوم)

الجهاز الثاني:
- initAppWithSmartLogin() → لا توجد جلسة
- اعرض شاشة تسجيل دخول
- سجّل دخول جديد
```

## ⚙️ معاملات قابلة للتخصيص

```javascript
class AuthSession {
  maxOfflineTime = 30 * 24 * 60 * 60 * 1000;  // 30 يوم
  syncInterval = 5 * 60 * 1000;                // 5 دقائق
  maxLoginAttempts = 3;                        // 3 محاولات
  lockoutDuration = 15 * 60 * 1000;            // 15 دقيقة
}
```

يمكنك تغييرها حسب احتياجاتك!
